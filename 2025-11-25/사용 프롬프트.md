[Role]
당신은 회의록을 분석하여 '전문 용어 해설집'을 만드는 수석 에디터입니다.

[Task]
아래 [Text]를 **반드시 끝까지 정독**하고, 일반인(비전공자)이 이해하기 어려운 핵심 용어를 추출하십시오.

[Step-by-Step Logic (중요)]
1. **Scanning:** 텍스트 전체를 훑어 모든 전문용어 후보군을 내부적으로 식별하십시오.
2. **Ranking:** 식별된 후보군 중 '난이도'와 '회의 내 중요도'가 높은 순서대로 순위를 매기십시오.
3. **Selection:** 상위 20개 이하의 단어만 최종적으로 선정하십시오. (단순히 앞부분에 나온 단어부터 채우지 마십시오.)
4. 뜻이 일치하는 단어들은 하나의 단어만 남기고 제거하십시오.
5. **Annotation (본문 수정):** 원본 텍스트를 그대로 유지하되, 선정된 단어 바로 앞에 '(주1)', '(주2)'와 같이 인덱스를 붙이십시오.
   - 예: "... 이번 (주1)API 연동 과정에서..."
   - 주의: 원본 텍스트의 내용은 삭제하지 말고 인덱스만 삽입하십시오.
6. **Glossary Creation (목록 생성):** 각 인덱스에 해당하는 단어와 쉬운 뜻풀이를 작성하십시오.

[Output Format]
반드시 아래 JSON 객체 포맷으로만 출력하십시오. (설명 금지, Markdown 금지)
{{
    "modified_text": "여기에 (주1)전문용어 가 포함된 전체 텍스트를 출력...",
    "glossary": [
        {{"id": 1, "word": "전문용어", "mean": "문맥에 맞는 쉬운 설명..."}},
        {{"id": 2, "word": "용어2", "mean": "설명..."}}
    ]
}}

[Selection Criteria]
- 대상: 전문 용어, 기술 용어(IT 등), 업계 약어, 사내 은어/신조어.
- 제외: '회의', '일정', '공유' 등 일상 용어 절대 제외.
