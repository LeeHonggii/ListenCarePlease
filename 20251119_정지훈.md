# 탐색한 모델에 1106회의록으로 성능 테스트
## 진행과정
1. 구글 드라이브 /content/drive/MyDrive/final_project/ 경로에 있는 1106_오후회의.json 파일 불러오기
2. 불러온 파일 loaded_data 라는 변수에 저장
3. GPT-4o-mini, GPT-4o 모델에 테스트
4. 시간이 명시 없을 때 일반 회사 업무 종료 시간인 18시로 설정
5. 결과물을 1106_오후회의_todo_list.json으로 저장

## 프롬포트
system_prompt = f"""
    당신은 전문적인 '회의록 분석 비서'입니다.
    제공된 회의록 텍스트를 분석하여 '실행 가능한 할 일(To-Do List)'을 JSON 형식으로 추출하세요.

    [현재 날짜]
    {today_date}

    [추출 규칙]
    1. 명확하게 담당자가 지정되고, 실행하기로 합의된 안건만 추출하세요.
    2. 날짜가 상대적일 경우(예: 내일, 다음 주 수요일) 현재 날짜를 기준으로 'YYYY-MM-DD' 형식으로 변환하세요.
    3. 모호하거나 거절된 요청은 제외하세요.
    4. 중요도(priority)는 문맥에 따라 'High', 'Medium', 'Low'로 판단하세요.

    [출력 형식 - JSON]
    {{
        "todos": [
            {{
                "task": "할 일 내용",
                "assignee": "담당자 이름",
                "due_date": "YYYY-MM-DD",
                "priority": "High/Medium/Low"
            }}
        ]
    }}
    """
