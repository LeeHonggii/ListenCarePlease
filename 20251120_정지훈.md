# 구글 캘린더 연동
## 구글 캘린더 API 생성
### 진행 과정
1.1 Google Cloud Console 접속.
1.2 [API 및 서비스] > [OAuth 동의 화면] 메뉴 이동.
1.3 External(외부) 선택 후 만들기.
1.4 앱 이름(예: My ToDo Service), 이메일 등 필수 정보만 입력하고 저장.
1.5 **[테스트 사용자]**에 본인 구글 이메일 추가 (앱 게시 전까지는 등록된 사람만 로그인 가능).
1.6 [사용자 인증 정보] > [+ 사용자 인증 정보 만들기] > [OAuth 클라이언트 ID].
1.7 애플리케이션 유형: 데스크톱 앱 (Colab이나 로컬 테스트용).
1.8 이름: ListenCarePlease
1.9 [만들기] 후 나오는 JSON 파일을 다운로드하여 **client_secret.json**으로 이름 변경 후 구글 드라이브(/content/drive/MyDrive/final_project/google-calendar)에 업로드.
1.10 화면 중앙의 '게시 상태'가 **"테스트(Testing)"**로 되어 있는지 확인 후 그 아래 [테스트 사용자 (Test users)] 항목을 찾습니다.
1.11 [+ ADD USERS] (사용자 추가) 버튼을 누릅니다.
1.12 본인의 구글 이메일 주소를 입력하고 **[저장]**을 누릅니다.

Temperature (창의성 설정): 현재 코드에 temperature=0.3으로 설정되어 있음. 이는 "10번 중 3번 정도는 덜 뻔한 단어를 선택해라"라는 의미와 비슷해서, 실행할 때마다 표현이나 추출하는 항목이 미세하게 달라질 수 있음
